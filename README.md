I built an expense tracker app with a simple goal in mind: to make managing day-to-day finances easier and more intuitive. The idea came from noticing how often people lose track of small, everyday expenses, which eventually add up and affect their overall budget. So, I designed this app to help users record both their expenses and income in a clean and organized way, giving them better control over their financial habits.

The app is developed using Java in Android Studio, which allowed me to create a smooth and responsive mobile experience. One of the key features is the ability to quickly add transactions. Users can enter details like the amount, category, and type (income or expense) within seconds. I focused on keeping the interface simple so that anyone can use it without confusion, even if they’re not very tech-savvy.

What makes this project more meaningful is the way it presents financial data. Instead of just listing numbers, the app uses interactive charts powered by AnyChart. These visualizations help users understand their spending patterns at a glance. For example, they can see their daily and monthly summaries, identify where most of their money is going, and adjust their habits accordingly. Seeing expenses in a visual format makes the whole experience more engaging and insightful.

Working on this project helped me strengthen my skills in Android development, especially in handling user input, managing data, and integrating third-party libraries like AnyChart. It also gave me a deeper appreciation for user experience design—understanding how small details can make a big difference in how people interact with an app.
